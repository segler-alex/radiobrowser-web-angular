You are owner of a stream

Stream information in this database can only be changed from the owner of the stream. To make sure you are the owner, you have to add special HTTP-headers to your stream. (HLS streams are currently not supported)

Main header

By default the default http-headers a stream has are only used to extract bitrate and online information. To enable overriding other information too (name, genre/tags, country,...) you have to set the special header icy-index-metadata to the value "1".

This will then force radio-browser.info to use the transmitted "name", "genre", "homepage" that your stream transmits. These fields should already be easily changeable by any audio streaming software.

You can check the information that has been extracted from your stream by searching it on this website and clicking It will show you a list of meta information extracted from your stream in the last 30 days.

From now on every change you make on your stream will automatically be picked up after 24 hours by radio-browser.info.

Other supported headers

icy-main-stream-url override the main stream url
icy-country-code override the country code
e.g.: AT
icy-logo override the logo/favicon url
e.g.: http://example.com/besticon.png
icy-geo-lat-long override the latitude and longitude of the stream location on a map
e.g.: 12.345,-23.456

Example: Icecast2 config

Full documentation at icecast.org 


<mount type="normal">
  <mount-name>/teststream</mount-name>

  <http-headers>
      <header name="icy-index-metadata" value="1" />
      <header name="icy-logo" value="https://example.com/besticon.png" />
      <header name="icy-country-code" value="at" />
      <header name="icy-main-stream-url" value="http://example.com:8000/teststream" />
      <header name="icy-geo-lat-long" value="12.345,-23.456" />
  </http-headers>
</mount>
